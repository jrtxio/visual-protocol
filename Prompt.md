# Visual UDS - UDS协议学习平台 需求文档

## 项目概述

Visual UDS 是一个交互式汽车诊断协议学习网站，旨在通过可视化动画和交互式演示帮助初学者学习UDS（统一诊断服务）协议。这是一个GitHub Pages项目，采用纯HTML5/CSS3/JavaScript技术栈，无需任何框架依赖。

## 项目结构

```
visual-uds/
├── index.html              # 主页面
├── docs/                   # GitHub Pages部署目录
│   ├── index.html          # 主页面（生产版本）
│   ├── manifest.json       # PWA配置文件
│   ├── sw.js              # Service Worker（离线支持）
│   └── assets/            # 静态资源
├── Prompt.md               # 原始需求文档
├── Prompt_Updated.md       # 本需求文档（更新版）
├── CLAUDE.md              # Claude开发指导文档
├── LICENSE                # Apache 2.0许可证
└── QWEN.md               # Qwen上下文文档
```

## 页面结构与功能

### 1. 基础概念页面 (基础概念)
- **目标**: 集中于文字讲解协议
- **内容**:
  - UDS协议核心概念详解
  - 客户端-服务器模型
  - 服务导向架构
  - 分层安全机制
  - 诊断数据管理
  - 协议栈结构
  - 诊断通信模型可视化
- **功能**:
  - 详细文字解释
  - 简单的通信模型可视化

### 2. 服务详解页面 (服务详解)
- **目标**: 详细服务信息
- **内容**:
  - 按功能单元分组的所有UDS服务 (20+ 服务)
  - 诊断和通信管理单元
  - 数据传输单元
  - 存储数据传输单元
  - 输入输出控制单元
  - 例行程序单元
  - 上传下载单元
- **功能**:
  - 详细服务信息展示
  - 服务ID、名称、描述
  - 支持子功能标识
  - 使用示例
  - 专业知识讲解

### 3. 交互演示页面 (交互演示)
- **目标**: 动画演示
- **内容**:
  - 诊断会话建立演示
  - 安全访问演示
  - 数据读取演示
  - 故障码演示
  - 协议栈可视化 (双栈)
  - 消息流向展示
- **功能**:
  - 交互式动画演示
  - 协议栈实时联动
  - 状态和日志显示
  - 双协议栈可视化

### 4. 实用工具页面 (实用工具)
- **目标**: 提供更多实用工具
- **内容**:
  - UDS消息解析器
  - 进制转换器 (十六进制、十进制、二进制)
  - 服务ID查询
  - DID (数据标识符) 查询
  - 例程ID查询
  - DTC (故障码) 解析器
  - 诊断会话计算器
- **功能**:
  - 消息解析
  - 格式转换
  - 信息查询
  - 状态计算

### 5. 知识测试页面 (知识测试)
- **目标**: 知识评估
- **功能**:
  - 多种题型 (单选、多选、判断、填空)
  - 自动评分
  - 详细解析
  - 学习进度跟踪

## 技术规格

### 前端技术
- **HTML5**: 语义化结构，支持PWA和可访问性
- **CSS3**: 现代化设计风格，渐变背景、毛玻璃效果、响应式布局
- **JavaScript**: 零外部依赖，实现所有交互逻辑

### 设计风格
- **色彩方案**: 蓝紫渐变主色调（#667eea → #764ba2），绿色强调色（#4CAF50）
- **UI组件**: 卡片式设计，圆角边框，阴影效果，渐变背景
- **动画效果**: 平滑过渡，微交互，悬停效果

### 功能模块
1. **基础概念**: 详尽的协议介绍
2. **服务详解**: 完整的UDS服务信息
3. **交互演示**: 动画演示协议流程
4. **知识测试**: 多样化的测试题型
5. **实用工具**: 专业工具集

## 开发与部署

### 本地开发
```bash
# 启动本地服务器
python -m http.server 8000
# 或使用其他静态文件服务器
npx serve .
```

### 部署方式
项目通过GitHub Pages进行部署，源码位于主分支，发布内容在`docs/`目录下。

### 测试要求
- 移动设备兼容性（320px-768px）
- 桌面浏览器兼容性（Chrome, Firefox, Safari, Edge）
- 动画性能保持60fps
- 无JavaScript错误
- 中文字符正确显示

## 关键依赖和限制

### 约束条件
- **纯静态内容**: 仅支持GitHub Pages部署，不允许服务器端代码
- **零外部依赖**: 禁止使用外部库或框架
- **中文界面**: 所有用户界面保持中文

### 浏览器兼容性
- Chrome 60+
- Firefox 55+
- Safari 12+
- Edge（最新版本）
